Parallel and Heterogeneous Computing with Microsoft PPL & AMP in C++

Beräkningsintensivt arbete utförs ofta bäst genom att flytta till parallell bearbetning, för att utnyttja flera kärnor. Dessutom börjar många utvecklare att se till att använda olika typer av hårdvara för att optimera programvaran ytterligare.

Idag ser många C ++ -programmerare på Microsoft PPL (Parallell Patterns Library) och Microsoft AMP (Accelerated Massive Parallelism) för att hjälpa dem skriva parallella algoritmer för att köras på både CPU och GPU. Microsoft PPL är utformat för att hjälpa utvecklare att snabbt få sin kod att köra på flera kärnor, medan AMP-biblioteken är avsedda att ta bort mycket av komplexiteten hos heterogen databehandling (särskilt kring blandarhårdvara från olika tillverkare). Kursen undersöker både PPL och AMP, men det har ett större fokus på den mer avancerade AMP-funktionaliteten.

Du kommer ut ur "Parallell och heterogen databehandling" och vet följande:

Hur man undviker vanliga optimeringsfall.
När man ska dra nytta av parallellitet.
Hur den underliggande hårdvaran bidrar till parallellitet.
Hur man utnyttjar flera kärnor med C ++ och Microsoft PPL.
Hur man utnyttjar GPU-enheten över flera tillverkare med C ++ och Microsoft AMP.
Hur man undviker vanliga parallella och heterogena datorfall.
Hur man debugger och gräver i parallella och heterogena program.
